  <action>
1. Create `cmd/history.go` with:
   ```go
   var historyCmd = &cobra.Command{
       Use:   "history",
       Short: "View version history for tracked policies",
       Long:  "Display version history for tracked Conditional Access policies.\nWithout --policy, shows all tracked policies with version counts.\nWith --policy, shows full version timeline with diff summaries.",
       RunE:  runHistory,
   }
   ```

2. In init(), register with rootCmd and add flags:
   - `--policy` (string): Policy slug to show detailed history for
   - `--json` (bool): Output in JSON format (table default for humans)

3. Implement `runHistory`:
   a. Load config, validate tenant (same pattern as status.go)
   b. Create git backend, read manifest
   c. If `--policy` is empty: **List all policies mode**
      - For each policy in manifest (sorted by slug), call backend.ListVersionTags to get count
      - Render table with columns: POLICY, VERSIONS, CURRENT VERSION, LAST DEPLOYED
      - For JSON: output array of {slug, version_count, current_version, last_deployed}
   d. If `--policy` is set: **Single policy history mode**
      - Validate slug exists in manifest; error if not tracked
      - Call backend.ListVersionTags(tenant, slug) for full history
      - Compute diff summaries for each version:
        * For each tag (newest first), read its blob and the previous version's blob via ReadTagBlob
        * ComputeDiff between consecutive versions (current tag as desired, previous tag as actual)
        * Summarize: "{N} fields changed: path1, path2, ..." (truncate paths at 3 + "...")
        * For the oldest version (no predecessor), show "initial import"
        * Performance note: policies rarely exceed ~20 versions per user decision, so computing all summaries upfront is acceptable
      - For table output: render with columns VERSION, DATE, CHANGES (the diff summary)
        Use text/tabwriter for alignment, same pattern as output.RenderHistory but with the added CHANGES column
      - For JSON output:
        ```json
        {
          "schema_version": 1,
          "slug": "policy-name",
          "history": [
            {"version": "1.2.0", "timestamp": "2026-03-05...", "message": "...", "changes": "3 fields changed: state, displayName, conditions.users"}
          ]
        }
        ```

4. Helper function `computeDiffSummaries(backend *state.GitBackend, tenant, slug string, tags []state.VersionTag) []string`:
   - Returns a string slice parallel to tags (same indices)
   - For i=len(tags)-1 (oldest): return "initial import"
   - For each other tag: read both blobs, unmarshal to map, ComputeDiff, build summary string
   - If ReadTagBlob fails for predecessor, return "unknown" for that entry
   - Summary format: collect diff paths, take top-level field names (split on ".", take first segment), deduplicate, join. "{N} fields changed: {fields}" or "no changes" if empty

5. The history command is purely read-only. No restore capability, no interactive TUI, no huh dependency. This is per user decision: "cactl history [--policy slug] as standalone read-only command for viewing version history without restore."
