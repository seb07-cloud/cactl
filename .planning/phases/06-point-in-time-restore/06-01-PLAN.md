Add interactive history browser and point-in-time restore to `cactl rollback -i`.

Purpose: Users can browse all tracked policies, select one, view its version history with diff summaries, pick a version, see the full diff against current desired state, and restore -- all through an arrow-key TUI wizard. 2. Create `internal/tui/selector.go` with:
   - `SelectPolicy(slugs []string) (string, error)` -- huh.NewSelect with arrow-key navigation, title "Select a policy"
   - `SelectVersion(tags []state.VersionTag, summaries []string) (string, error)` -- huh.NewSelect where each option label shows "version  date  summary" (e.g. "1.2.0  2026-03-05  3 fields changed: state, displayName"), value is the version string
   - `ConfirmRestore(slug, version string) (bool, error)` -- huh.NewConfirm with "Restore {slug} to {version}? [y/N]", default false
   - `ConfirmOverwrite(filePath string) (bool, error)` -- huh.NewConfirm with warning about uncommitted changes, "Overwrite {filePath} with uncommitted changes? [y/N]", default false
   - `SelectAction() (string, error)` -- huh.NewSelect with options "Restore this version" / "Back to versions" / "Quit", returns the chosen action string. This enables back-navigation from the diff view per the loop pattern from research.

3. Create `internal/tui/restore.go` with:
   - `RestoreConfig` struct holding: `Backend *state.GitBackend`, `Tenant string`, `Manifest *state.Manifest`, `UseColor bool`, `RepoDir string`
   - `RunInteractiveRestore(ctx context.Context, cfg RestoreConfig) error` -- the main wizard loop:
     a. Build sorted slug list from cfg.Manifest.Policies
     b. Call SelectPolicy to pick a policy
     c. Call backend.ListVersionTags to get history
     d. For each version, compute diff summary by reading current version blob and previous version blob via ReadTagBlob, then reconcile.ComputeDiff, then output.DiffSummary. For the oldest version, show "initial version". Lazy: only compute summaries for the tags returned (policies rarely exceed ~20 versions per user decision).
     e. Call SelectVersion to pick a version
     f. Read historical JSON via backend.ReadTagBlob
     g. Read current desired state via cmd.ReadDesiredPolicies(cfg.Tenant) -- get the selected slug's Data map
     h. Compute diff: reconcile.ComputeDiff(historicalMap, currentDesiredMap) -- historical is "desired" (what we want), current is "actual" (what we have). This matches ComputeDiff convention.
     i. If no diffs, print "Selected version matches current desired state" and loop back to version selection
     j. Render diffs using output.RenderFieldDiffs (new exported function, see below)
     k. Call SelectAction for "Restore / Back / Quit"
     l. If "Back", loop back to version selection (step e)
     m. If "Quit", return nil
     n. If "Restore":
        - Check uncommitted changes: run `git status --porcelain policies/<tenant>/<slug>.json` via os/exec
        - If dirty, call ConfirmOverwrite; if declined, loop back
        - Write file via cmd.WritePolicyFile(tenant, slug, historicalJSON)
        - Compute new patch version: parse current manifest entry version, bump patch
        - Auto-commit: `git add policies/<tenant>/<slug>.json` then `git commit -m "restore: <slug> to v<version>"` via os/exec
        - Print success message
        - Auto-run plan: call cmd.RunPlanFromRestore(ctx) which invokes the plan command programmatically (see task 2)
        - Return nil

   Note: Import the `cmd` package would create a circular dependency (tui imports cmd, cmd imports tui). Instead:
   - Pass `WritePolicyFile func(tenant, slug string, data []byte) error` as a field in RestoreConfig
   - Pass `ReadDesiredPolicies func(tenant string) (map[string]map[string]interface{}, error)` as a field in RestoreConfig (simplified return type -- just slug->data map, not BackendPolicy)
   - Pass `RunPlan func(ctx context.Context) error` as a field in RestoreConfig
   This avoids circular deps and keeps tui testable with injected functions.

4. Add `DiffSummary(diffs []reconcile.FieldDiff) string` to `internal/output/diff.go`:
   - If len(diffs) == 0, return "no changes"
   - Collect unique top-level path segments (split on "." take first element) to summarize at field group level
   - Return "{N} fields changed: {path1}, {path2}, ..." (truncate to 3 paths + "..." if more)

5. Add `RenderFieldDiffs(w io.Writer, diffs []reconcile.FieldDiff, useColor bool)` to `internal/output/diff.go`:

   - This allows tui/restore.go to render diffs without accessing the unexported renderFieldDiff

       // ... error handling ...

       restoreCfg := tui.RestoreConfig{
           Backend:  backend,
           Tenant:   cfg.Tenant,
           Manifest: manifest,
           UseColor: output.ShouldUseColor(viper.GetViper()),
           RepoDir:  ".",
           WritePolicyFile: func(tenant, slug string, data []byte) error {
               return WritePolicyFile(tenant, slug, data)
           },
           ReadDesiredPolicies: func(tenant string) (map[string]map[string]interface{}, error) {
               policies, err := ReadDesiredPolicies(tenant)
               if err != nil { return nil, err }
               result := make(map[string]map[string]interface{}, len(policies))
               for slug, bp := range policies {
                   result[slug] = bp.Data
               }
               return result, nil
           },
           RunPlan: func(ctx context.Context) error {
               return runPlan(planCmd, nil)
           },
       }
       return tui.RunInteractiveRestore(ctx, restoreCfg)
   }
