RED: Create `internal/semver/version_test.go` with table-driven tests:

**DetermineBump tests:**
1. Field in major_fields list -> BumpMajor (e.g., "conditions.users.includeUsers" changed)
2. Field matching major_fields prefix -> BumpMajor (e.g., "state" matches "state" exactly)
3. Field in minor_fields list -> BumpMinor (e.g., "grantControls.builtInControls" matches "grantControls" prefix)
4. Field not in major or minor -> BumpPatch (e.g., "displayName")
5. Mixed: one major field + one patch field -> BumpMajor (highest wins, short-circuit)
6. Mixed: one minor field + one patch field -> BumpMinor
7. Empty diffs -> BumpPatch
8. Prefix matching: "conditions" trigger matches "conditions.users.includeGroups"

**BumpVersion tests:**
1. "1.0.0" + BumpPatch -> "1.0.1"
2. "1.0.0" + BumpMinor -> "1.1.0" (patch resets to 0)
3. "1.0.0" + BumpMajor -> "2.0.0" (minor and patch reset to 0)
4. "1.2.3" + BumpMinor -> "1.3.0"
5. "2.5.9" + BumpMajor -> "3.0.0"
6. Invalid version "abc" -> error

Run tests: `go test ./internal/semver/ -v` (MUST FAIL)

- `BumpVersion(current string, level BumpLevel) (string, error)` - parse "X.Y.Z", increment appropriate component, reset lower. Return formatted string.

**checkConflictingConditions (VALID-03):**
1. Same GUID in includeUsers and excludeUsers -> SeverityError
2. Same GUID in includeGroups and excludeGroups -> SeverityError
3. No overlap -> no result
4. Same app in includeApplications and excludeApplications -> SeverityError

**checkEmptyIncludes (VALID-04):**
1. includeUsers empty, includeGroups empty, includeRoles empty -> SeverityWarning
2. includeUsers has "All" -> no result (not empty)

**checkOverlyBroad (VALID-05):**
1. includeUsers contains "All", excludeUsers empty, excludeGroups empty, state "enabled" -> SeverityWarning
2. includeUsers contains "All" but has excludeGroups -> no result
3. state "disabled" even with All and no exclusions -> no result (disabled is safe)
