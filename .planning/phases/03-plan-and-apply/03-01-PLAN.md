2. **Update**: backend has "ca-mfa", manifest has entry with LiveObjectID, live map has that ID with different JSON -> ActionUpdate with non-empty Diff
3. **Noop**: backend has "ca-mfa", manifest has entry, live has matching JSON -> no action emitted (PLAN-09 idempotency)
4. **Recreate**: backend has "ca-mfa", manifest has entry with LiveObjectID "abc123", but live map does NOT contain "abc123" (ghost) -> ActionRecreate
5. **Untracked**: live has policy ID "xyz789" not tracked in manifest -> ActionUntracked
6. **Mixed**: backend has 2 policies (one create, one update), live has 1 untracked -> 3 actions total
7. **Empty both**: no backend, no live -> empty actions slice

The Reconcile function signature: `Reconcile(backend map[string]BackendPolicy, live map[string]LivePolicy, manifest *state.Manifest) []PolicyAction`

Define `BackendPolicy` struct: `Data map[string]interface{}` (the normalized JSON).
Define `LivePolicy` struct: `NormalizedData map[string]interface{}`, `Slug string` (derived from display name).

Run tests: `go test ./internal/reconcile/ -run TestReconcile -v` (MUST FAIL)

GREEN: Implement `internal/reconcile/engine.go`:
1. Iterate backend policies. For each slug:
   - If not in manifest -> ActionCreate
   - If in manifest but LiveObjectID not in live map -> ActionRecreate

2. Find untracked: build set of all tracked LiveObjectIDs from manifest. Iterate live map; any ID not in tracked set -> ActionUntracked.
3. Return sorted actions (by slug) for deterministic output.
