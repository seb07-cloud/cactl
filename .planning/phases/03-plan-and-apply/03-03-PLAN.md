Extend `internal/graph/policies.go` with three new methods on *Client:

1. `CreatePolicy(ctx context.Context, policyJSON map[string]interface{}) (string, error)` - POST to `/identity/conditionalAccess/policies`, marshal policyJSON to body, expect 201 Created, decode response to extract `id` field, return new ID.

2. `UpdatePolicy(ctx context.Context, id string, policyJSON map[string]interface{}) error` - PATCH to `/identity/conditionalAccess/policies/{id}`, marshal policyJSON to body, expect 204 No Content. Send full body (not partial) per research anti-pattern guidance.

3. `DeletePolicy(ctx context.Context, id string) error` - DELETE to `/identity/conditionalAccess/policies/{id}`, expect 204 No Content.

All methods: use `c.do()` for authenticated requests, set Content-Type application/json. On non-expected status codes, read body and return descriptive error with status code and body.

Note: The `c.do()` method currently builds the URL from the path argument directly. Check whether it expects full URL or path-only. Looking at ListPolicies: it passes `c.baseURL + "/identity/..."` as the full URL. So write methods must also prepend c.baseURL.

Create `internal/graph/batch.go`:
- `BatchRequestItem` struct: ID string `json:"id"`, Method string `json:"method"`, URL string `json:"url"`
- `BatchResponseItem` struct: ID string `json:"id"`, Status int `json:"status"`, Body json.RawMessage `json:"body"`
- `BatchRequest` struct: Requests []BatchRequestItem `json:"requests"`
- `BatchResponse` struct: Responses []BatchResponseItem `json:"responses"`
- `ExecuteBatch(ctx context.Context, requests []BatchRequestItem) ([]BatchResponseItem, error)` - POST to `c.baseURL + "/$batch"` with batch JSON body. Note: Graph batch endpoint is `https://graph.microsoft.com/v1.0/$batch`. Return decoded responses.

Add tests to `internal/graph/client_test.go`:
- TestCreatePolicy: httptest server returns 201 with `{"id": "new-id-123"}`, verify returned ID
- TestCreatePolicy_Error: httptest server returns 400, verify error message contains status code
- TestUpdatePolicy: httptest server returns 204, verify no error and correct PATCH method/path
- TestDeletePolicy: httptest server returns 204, verify no error and correct DELETE method/path
- TestExecuteBatch: httptest server returns batch response with 2 items, verify decoded responses

Run: `go test ./internal/graph/ -v`

Create `internal/resolve/resolver.go`:

- `ObjectRef` struct: ID string, Type string (one of: "user", "group", "namedLocation", "servicePrincipal")
- `Resolver` struct: graphClient (need a minimal interface, not concrete *graph.Client, for testability), cache map[string]string, mu sync.Mutex
- Define `BatchClient` interface: `ExecuteBatch(ctx context.Context, requests []graph.BatchRequestItem) ([]graph.BatchResponseItem, error)` -- allows mock injection in tests
- `NewResolver(client BatchClient) *Resolver`
- `ResolveAll(ctx context.Context, refs []ObjectRef) error`:
  1. Filter out refs whose ID is already in cache
  2. Deduplicate remaining refs by ID
  3. Build batch request items: map ObjectRef.Type to Graph API URL:
     - "user" -> `/users/{id}?$select=id,displayName`
     - "group" -> `/groups/{id}?$select=id,displayName`
     - "namedLocation" -> `/identity/conditionalAccess/namedLocations/{id}`
     - "servicePrincipal" -> `/servicePrincipals/{id}?$select=id,displayName`
  4. Chunk into batches of 20 (Graph API limit)
  5. For each batch: call ExecuteBatch, parse responses
  6. For each response: if status 200, decode body for "displayName" field, cache it. If 404, cache "{id} (deleted)". If other error, cache the raw ID (graceful degradation).
- `DisplayName(id string) string` - lock mutex, return cached name or raw ID as fallback
- `CollectRefs(actions []reconcile.PolicyAction) []ObjectRef` - scan policy JSON for known GUID fields (conditions.users.includeUsers, excludeUsers, includeGroups, excludeGroups, conditions.locations.includeLocations, excludeLocations, conditions.applications.includeApplications, excludeApplications) and return ObjectRef slice with correct types

Create `internal/resolve/resolver_test.go` with tests:
1. ResolveAll with mock batch client returning 200 for group and user -> DisplayName returns display names
2. ResolveAll with 404 for deleted group -> DisplayName returns "{id} (deleted)"
3. ResolveAll with already-cached ID -> no batch call made (verify mock not called for cached)
4. CollectRefs extracts GUIDs from PolicyAction BackendJSON with correct types
5. DisplayName for unknown ID -> returns raw ID
6. Batching: 25 refs -> two batch calls (20 + 5)

Import `github.com/seb07-cloud/cactl/internal/graph` for batch types.
Import `github.com/seb07-cloud/cactl/internal/reconcile` for PolicyAction.

Run: `go test ./internal/resolve/ -v`
