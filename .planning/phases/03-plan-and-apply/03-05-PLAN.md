Create `cmd/apply.go`:

Register `applyCmd` on rootCmd in init(). Command: `cactl apply`.
Flags:
- `--auto-approve` bool (default false): skip confirmation prompt (PLAN-06)
- `--dry-run` bool (default false): generate plan but make no writes (PLAN-07)

`runApply(cmd *cobra.Command, args []string) error` flow:

**Phase A: Generate plan (reuse plan logic)**
1. Load config, validate tenant, create clients (same as plan command)
2. Load backend policies, live policies, manifest (same as plan command)
3. Reconcile to get actions
4. Compute semver bumps for updates
5. Run validations -- if any SeverityError results, render them and return ExitError code 3 (block apply)

**Phase B: Display plan**
6. Render plan (same as plan command -- human or JSON based on --output)
7. Filter to actionable items (Create, Update, Recreate). If empty: print "No changes. Infrastructure is up-to-date." and return nil (exit 0, PLAN-09)

**Phase C: Dry-run check (PLAN-07)**
8. If --dry-run: print "Dry run complete. No changes applied." and return nil

**Phase D: Confirmation (PLAN-05, PLAN-08)**
9. CI mode check: if cfg.CI and !autoApprove, return ExitError code 2 "ci mode requires --auto-approve for write operations"
10. If !autoApprove:
    a. Standard confirmation: `confirm("Do you want to apply these changes? [Y/n]: ")` using bufio.Scanner. Empty or "y"/"yes" = proceed.
    b. If any ActionRecreate in actionable: escalated confirmation `confirmExplicit("Recreate actions will DELETE and re-CREATE policies. Type 'yes' to confirm: ")` -- must type exactly "yes" (PLAN-08).
    c. If either confirmation denied: print "Apply cancelled." and return nil

**Phase E: Execute actions**
11. For each actionable action (sorted by slug for deterministic order):
    - **ActionCreate**: call `graphClient.CreatePolicy(ctx, action.BackendJSON)` to get new ID. Compute version "1.0.0". Write policy to backend with `backend.WritePolicy()`. Create version tag with `backend.CreateVersionTag()`. Update manifest entry with new LiveObjectID, version, timestamp, deployer, auth mode, backend SHA. Write manifest.
    - **ActionUpdate**: call `graphClient.UpdatePolicy(ctx, action.LiveObjectID, action.BackendJSON)`. Compute new version via BumpVersion. Write policy to backend. Create version tag. Update manifest entry. Write manifest.
    - **ActionRecreate**: call `graphClient.CreatePolicy(ctx, action.BackendJSON)` (ghost case: old is already deleted, just create new). Compute version from current + bump. Write policy to backend. Create version tag. Update manifest with new LiveObjectID. Write manifest.
12. For each action, print success line: "Applied: + ca-slug (1.0.0)" or "Applied: ~ ca-slug (1.0.0 -> 1.1.0)"
13. On error mid-apply: print which policy failed, how many succeeded, and return ExitError code 2. State is consistent up to the last successful action (per-action state updates).

**Phase F: Summary**
14. Print "Apply complete: N created, N updated, N recreated."
15. Return nil (exit 0)

**SEMV-05 (user override)**: For v1, the suggested bump is auto-applied. User override is deferred to a future flag (--bump-level). Add a comment: `// TODO: SEMV-05 --bump-level flag for user override`.

**Helper functions (private to cmd/apply.go):**
- `confirm(prompt string) bool` - bufio.Scanner, accept empty/y/yes
- `confirmExplicit(prompt string) bool` - bufio.Scanner, accept only "yes"
- `filterActionable(actions []reconcile.PolicyAction) []reconcile.PolicyAction` - filter out Noop and Untracked
- `hasAction(actions []reconcile.PolicyAction, t reconcile.ActionType) bool`
- `deployerIdentity(cfg *types.Config) string` - "cactl/{auth-mode}" for manifest deployed_by field

**Populate manifest Entry fields:**
- Slug, Tenant from action + config
- LiveObjectID from Graph response (create) or existing (update)
- Version from semver computation
- LastDeployed: time.Now().UTC().Format(time.RFC3339)
- DeployedBy: deployerIdentity(cfg)
- AuthMode: cfg.Auth.Mode
- BackendSHA: from WritePolicy return value
