  <name>Task 1: GoReleaser config and version injection</name>
  <files>
    .goreleaser.yaml
    main.go
  </files>
  <action>
1. Create `.goreleaser.yaml` (v2 format) with:
   - `version: 2`, `project_name: cactl`
   - `before.hooks: [go mod tidy]`
   - Builds: `CGO_ENABLED=0`, goos `[linux, darwin, windows]`, goarch `[amd64, arm64]`, ignore `windows/arm64`
   - ldflags: `-s -w -X main.version={{.Version}} -X main.commit={{.Commit}} -X main.date={{.CommitDate}}`
   - flags: `-trimpath`
   - Archives: `tar.gz` default, `zip` for windows, name template `{{ .ProjectName }}_{{ .Version }}_{{ .Os }}_{{ .Arch }}`
   - Checksum: `checksums.txt`
   - Changelog: sort `asc`, groups for Features (`^feat`), Bug Fixes (`^fix`), Others (order 999). Exclude `^docs:`, `^test:`, `^chore\(deps\):`
   - Release: `github.owner: seb07-cloud, github.name: cactl`

2. In `main.go`, add version variables at package level (after imports):
   ```go
   var (
       version = "dev"
       commit  = "none"
       date    = "unknown"
   )
   ```
   These are populated by GoReleaser ldflags at build time. Add a `rootCmd.Version = version` line or pass version info to the root command setup. The simplest approach: add `cmd.SetVersionInfo(version, commit, date)` and create that function in `cmd/root.go` to set `rootCmd.Version`.

3. In `cmd/root.go`, add:
   ```go
   func SetVersionInfo(v, c, d string) {
       rootCmd.Version = fmt.Sprintf("%s (commit: %s, built: %s)", v, c, d)
   }
   ```
   Call `cmd.SetVersionInfo(version, commit, date)` in `main.go` before `cmd.Execute()`.
  </action>
  <verify>
    `cd /Users/seb/Git/cactl && go build -ldflags "-X main.version=test -X main.commit=abc -X main.date=now" -o cactl . && ./cactl --version` outputs version info. Clean up: `rm cactl`.
  </verify>
  <done>
    GoReleaser config produces 5 platform binaries with version injection. `cactl --version` displays version, commit, and build date.
  </done>

   - Permissions: `contents: write`

3. Create `examples/github-actions/cactl-plan.yml` (CICD-04):
   - Name: "cactl plan"
   - Triggers: pull_request on `policies/**`
   - Permissions: `id-token: write`, `contents: read`
   - Environment: `production`
   - Steps: checkout (fetch-depth 0), `azure/login@v2` with OIDC (client-id, tenant-id, subscription-id from vars), install cactl from latest release, `cactl plan --ci --tenant ${{ vars.AZURE_TENANT_ID }}`

4. Create `examples/github-actions/cactl-drift.yml` (CICD-06):
   - Name: "Scheduled Drift Check"
   - Triggers: `schedule: [{cron: '0 6 * * *'}]`, `workflow_dispatch`
   - Permissions: `id-token: write`, `contents: read`, `issues: write`
   - Steps: checkout, azure/login, install cactl, run `cactl drift --ci --tenant ${{ vars.AZURE_TENANT_ID }}`, on failure step create GitHub issue with `gh issue create` using `if: failure()`
