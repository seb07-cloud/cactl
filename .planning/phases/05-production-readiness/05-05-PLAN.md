# Plan 05-05: Enforce CI Coverage Threshold

## Gap Source

From 05-VERIFICATION.md: QUAL-03 partial — CI workflow collects coverage but `|| true` prevents enforcement. No step compares against 80% threshold.

## Objective

Add a coverage enforcement step to `.github/workflows/ci.yml` that fails the build when total test coverage drops below 80%.

## Tasks

<task id="1" autonomous="true">
**Edit `.github/workflows/ci.yml`** — Replace the coverage summary step

1. Remove `|| true` from the coverage summary step
2. Add a new step "Enforce coverage threshold" after the summary that:
   - Extracts total coverage percentage from `go tool cover -func=cover.out`
   - Compares against 80% threshold
   - Exits non-zero with a clear message if below threshold

Target YAML for the new step:
```yaml
- name: Enforce coverage threshold
  run: |
    COVERAGE=$(go tool cover -func=cover.out | grep "^total:" | awk '{print $3}' | tr -d '%')
    echo "Total coverage: ${COVERAGE}%"
    if awk "BEGIN {exit !(${COVERAGE}+0 < 80)}"; then
      echo "::error::Coverage ${COVERAGE}% is below 80% threshold"
      exit 1
    fi
```
</task>

- [ ] The step parses total coverage and compares against 80
