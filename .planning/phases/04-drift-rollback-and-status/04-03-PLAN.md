Create `cmd/rollback.go`:

Register `rollbackCmd` on rootCmd in init(). Command: `cactl rollback`.
Flags:
- `--policy` string (required): policy slug to roll back
- `--version` string (required): semver version to restore (e.g., "1.0.0")
- `--auto-approve` bool (default false): skip confirmation prompt

`runRollback(cmd *cobra.Command, args []string) error` flow:

1. Load config, validate tenant (same pattern as plan/apply/drift)
2. Get --policy and --version flag values. Validate both are non-empty, return ExitError code 3 if missing.
3. Create auth factory and credential
4. Create graph client
5. Create git backend: `state.NewGitBackend(".")`
6. Load manifest: `state.ReadManifest(backend, cfg.Tenant)`

**Step 7: Validate policy is tracked (ROLL-01)**
7. Check `manifest.Policies[slug]` exists. If not, return ExitError code 2: "policy '%s' is not tracked in the manifest. Run 'cactl import' first."

**Step 8: Read historical version from tag (ROLL-01)**
8. Call `backend.ReadTagBlob(cfg.Tenant, slug, version)`. If error:
   a. List available versions: `backend.ListVersionTags(cfg.Tenant, slug)`
   b. Print "Version %s not found for policy %s."
   c. If versions exist: print "Available versions:" followed by each version with timestamp
   d. Return ExitError code 2

**Step 9: Get current live state**
9. Get the LiveObjectID from manifest entry. Call `graphClient.GetPolicy(ctx, entry.LiveObjectID)`.
   - If error (e.g., 404 -- policy deleted out-of-band): print warning "Live policy not found (may have been deleted). Rollback would require recreating the policy." Return ExitError code 2 with message suggesting `cactl apply` instead. (Research pitfall 5)

**Step 10: Compute diff (ROLL-02)**
10. Unmarshal tag JSON to `map[string]interface{}` (desired = historical version). Normalize live policy via `normalize.Normalize(livePolicy.RawJSON)`, unmarshal to map (actual = current live). Call `reconcile.ComputeDiff(desired, actual)` to get field-level diff.

11. If diff is empty: print "No changes needed. Live policy already matches version %s." Return nil (exit 0).

**Step 11: Display diff and confirm (ROLL-02)**
12. Print header: "Rolling back %s to version %s"
13. Print field-level diff (reuse diff rendering from output package or inline: for each FieldDiff, print +/- with path and values). Use `output.RenderPlan` if it accepts a single-action slice, or render manually:
    - For DiffAdded: green `+ path: newValue`
    - For DiffRemoved: red `- path: oldValue`
    - For DiffChanged: yellow `~ path: oldValue -> newValue`
14. If `cfg.Output == "json"`: render as JSON with diff details, skip confirmation, return.

**Step 12: Confirmation**
15. If !autoApprove and !cfg.CI:
    - Prompt: "Apply rollback? [Y/n]: " (standard confirmation, same as apply)
    - If denied: print "Rollback cancelled." Return nil.
16. If cfg.CI and !autoApprove: return ExitError code 2 "ci mode requires --auto-approve"

**Step 13: Execute rollback (ROLL-03)**
17. Marshal tag JSON map back to bytes for PATCH body. Call `graphClient.UpdatePolicy(ctx, entry.LiveObjectID, tagMap)`.
    - On error: return ExitError code 2 with message.

**Step 14: Create new version (ROLL-04 -- never modify existing tags)**
18. Determine current version from manifest entry.Version. Compute PATCH bump: `bumpPatchVersion(currentVersion)` (always PATCH for rollback per research recommendation, satisfies SEMV-05 since rollback is a deployment event with auto-computed version).
19. Write policy blob to backend: `backend.WritePolicy(cfg.Tenant, slug, tagJSONBytes)` -- gets new blobHash.
20. Create NEW annotated tag: `backend.CreateVersionTag(cfg.Tenant, slug, newVersion, blobHash, message)` where message = "cactl rollback: <slug> <newVersion> (rolled back to <requestedVersion>)".
21. Update manifest entry: Version=newVersion, LastDeployed=now (RFC3339), DeployedBy=deployerIdentity(cfg), BackendSHA=blobHash. Write manifest.

**Step 15: Success output**
22. Print: "Rollback complete: %s rolled back to %s (new version: %s)"
23. Return nil (exit 0)

**Helper functions (private to cmd/rollback.go):**
- `bumpPatchVersion(version string) (string, error)` -- parse "MAJOR.MINOR.PATCH", return "MAJOR.MINOR.(PATCH+1)". Reuse from import command if available, or implement: split on ".", parse last component, increment, rejoin. Check if `internal/normalize/slug.go` or import command has this -- the import command uses `bumpPatchVersion` in cmd/import.go, so extract or duplicate the helper.
- `deployerIdentity(cfg *types.Config) string` -- "cactl/<auth-mode>", same as apply command. If apply already has this as a shared helper, import it; otherwise duplicate.
