Create `cmd/drift.go`:

Register `driftCmd` on rootCmd in init(). Command: `cactl drift`.
Flags:
- `--policy` string (optional): filter drift output to a single policy slug

`runDrift(cmd *cobra.Command, args []string) error` flow:

**Identical setup to plan command (steps 1-9 from cmd/plan.go):**
1. Load config via `config.LoadFromGlobal()`
2. Validate tenant is set. If not, return ExitError code 2 with message "tenant is required: use --tenant or set CACTL_TENANT"
3. Create auth factory and credential
4. Create graph client
5. Create git backend: `state.NewGitBackend(".")`
6. Load backend policies: iterate `backend.ListPolicies(cfg.Tenant)`, for each slug read policy, unmarshal to map
7. Load live policies: `graphClient.ListPolicies(ctx)`, normalize each, build map keyed by ID
8. Load manifest: `state.ReadManifest(backend, cfg.Tenant)`
9. Reconcile: `reconcile.Reconcile(backendPolicies, livePolicies, manifest)`

**Drift-specific logic (DIFFERS from plan -- NO semver, NO validation writes, NO resolver):**

10. Filter actions to actionable (non-noop). If `--policy` flag is set, further filter to only the matching slug.

11. If no actionable items:
    - Print success message: "No drift detected. Backend and live state are in sync."
    - Return nil (exit 0, DRIFT-03)

12. Render drift output:
    - If `cfg.Output == "json"`: call `output.RenderPlanJSON(os.Stdout, actions, nil, nil)` (DISP-05 -- JSON output for drift). The plan JSON schema works for drift since actions are the same type.
    - Else: call `output.RenderPlan(os.Stdout, actions, nil, nil, useColor)` (reuse plan renderer -- same sigils, same format)

13. Print remediation suggestions footer (DRIFT-04):
    ```
    Remediation options:
      cactl apply            apply backend state to live tenant
      cactl import --force   update backend from live tenant state
      (no action)            this was a report-only check
    ```

14. Return `&types.ExitError{Code: types.ExitChanges, Message: "drift detected"}` (exit 1, DRIFT-03)

**VALID-02 note:** The drift command validates that the reconciliation can be computed (auth works, Graph API reachable, backend readable). If any of these fail, return ExitError code 2. This satisfies VALID-02 (runtime validation of drift operations).

**Error handling:** Wrap any error in steps 1-9 with ExitError code 2 (fatal error).

**Key difference from plan:** No semver computation, no plan-time validations (break-glass etc.), no display name resolution (keeps drift fast for CI). The reconcile engine output is sufficient.
