Create `cmd/status.go`:

Register `statusCmd` on rootCmd in init(). Command: `cactl status`.
Flags:
- `--history` string (optional): show version history for a specific policy slug

`runStatus(cmd *cobra.Command, args []string) error` flow:

1. Load config via `config.LoadFromGlobal()`
2. Validate tenant is set. Return ExitError code 2 if missing.
3. Create git backend: `state.NewGitBackend(".")`
4. Load manifest: `state.ReadManifest(backend, cfg.Tenant)`

**History mode (if --history flag set):**
5. Call `backend.ListVersionTags(cfg.Tenant, historySlug)`.
   - If empty: print "No version history for policy '%s'." Return nil.
   - Call `output.RenderHistory(os.Stdout, historySlug, tags, useColor)` (or `RenderStatusJSON` variant for JSON output)
   - Return nil

**Normal status mode:**
6. If manifest has no policies: print "No tracked policies. Run 'cactl import' to get started." Return nil.

7. Attempt sync check (graceful degradation per research recommendation):
   a. Try to create auth factory and credential. If auth fails, set `syncAvailable = false`, print warning "Could not authenticate -- sync status will show as 'unknown'."
   b. If auth succeeds, create graph client, call `graphClient.ListPolicies(ctx)`.
   c. If ListPolicies fails (network error), set `syncAvailable = false`, print warning.
   d. If ListPolicies succeeds, build `liveIndex` map: key=policy ID, value=normalized JSON bytes. Use `normalize.Normalize()` on each policy's RawJSON, then compute SHA via `sha256.Sum256()`.

8. Build `[]types.PolicyStatus` from manifest entries. For each entry in manifest.Policies:
   - Slug, Version, LastDeployed, DeployedBy from entry
   - LiveObjectID from entry
   - SyncStatus: if !syncAvailable -> "unknown"
     - If entry.LiveObjectID not found in liveIndex -> "missing"
     - If liveIndex SHA matches entry.BackendSHA -> "in-sync"
     - Else -> "drifted"
   Sort entries by slug for deterministic output.

9. Build StatusSummary by counting each sync status.

10. Render:
    - If `cfg.Output == "json"`: build StatusOutput{SchemaVersion: 1, Tenant: cfg.Tenant, Policies: entries, Summary: summary}, call `output.RenderStatusJSON(os.Stdout, statusOutput)`
    - Else: call `output.RenderStatus(os.Stdout, entries, useColor)`

11. Return nil (exit 0). Status always exits 0 even if drift is detected (status is informational, not a gate).

**Performance note (research pitfall 4):** Use ListPolicies() once and index by ID for O(1) lookup per tracked policy, not GetPolicy() per entry.

**Sync SHA comparison:** The manifest's BackendSHA is the git object hash of the normalized policy JSON written during apply/import. To compare: normalize the live policy JSON, compute SHA-256 of the result, compare against entry.BackendSHA. IMPORTANT: The BackendSHA in the manifest is a git SHA (SHA-1 of "blob <size>\0<content>"), not a raw SHA-256. To compare correctly, either:
  - Option A: Normalize live JSON, write to a temp git hash-object command (same as backend.hashObject), compare SHA. This is the correct approach since it matches the git SHA format.
  - Option B: Use the backend.hashObject approach by passing normalized bytes through `git hash-object --stdin` and comparing the result to BackendSHA.
  Use option B by calling a helper that runs `git hash-object --stdin` on the normalized bytes and returns the SHA string. Add a public `HashObject(data []byte) (string, error)` method to GitBackend that wraps the existing private `hashObject`.
