GREEN: Add to `internal/state/backend.go`:

Define `VersionTag` struct at package level:
```go
type VersionTag struct {
    Version   string
    Timestamp string
    Message   string
}
```

Add `ListVersionTags(tenantID, slug string) ([]VersionTag, error)` method on *GitBackend:
- Build prefix: `refs/tags/cactl/<tenantID>/<slug>/`
- Run: `git for-each-ref --format=%(refname:strip=4)\t%(creatordate:iso)\t%(contents:lines=1) --sort=-version:refname <prefix>`
- `strip=4` removes "refs/tags/cactl/<tenant>/" leaving "<slug>/<version>"
- Parse tab-separated output. For each line, split on tab to get [refname, timestamp, message]. Extract version as last path component after slug. Build VersionTag.
- If output is empty, return empty slice (not nil, not error).

Add `ReadTagBlob(tenantID, slug, version string) ([]byte, error)` method on *GitBackend:
- Build tag name: `cactl/<tenantID>/<slug>/<version>`
- Run: `git cat-file blob <tagName>^{}`
- The `^{}` dereferences annotated tag object to the underlying blob.
- Return output bytes. On error, wrap with descriptive message including tag name.

6. **TestHashObject**: Write known data bytes, call HashObject. Verify it returns the same SHA as `git hash-object --stdin` would for those bytes. Also verify it matches the SHA returned by WritePolicy for the same content (since WritePolicy uses the private hashObject internally).

GREEN (continued): Add `HashObject(data []byte) (string, error)` public method on *GitBackend:

- Returns the git SHA-1 hash string for the given data bytes.
- Purpose: Status command needs this to compare manifest BackendSHA against live normalized JSON.
