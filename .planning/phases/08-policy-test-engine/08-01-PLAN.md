Create `internal/testengine/types.go` with all types needed by the test engine:

- `TestSpec` struct: Name, Description string, Policies []string (optional filter), Scenarios []Scenario. Use yaml struct tags.

- `SignInContext` struct: User string, Groups []string, Roles []string, Application string, ClientAppType string, Platform string, Location string, SignInRiskLevel string, UserRiskLevel string. All with yaml tags, optional fields with `omitempty`.
- `ExpectedOutcome` struct: Result string (block/grant/notApplicable), Controls []string (optional), SessionControls map[string]interface{} (optional).
- `PolicyWithSlug` struct: Slug string, Data map[string]interface{}.
- `EvalResult` int enum: ResultNotApplicable, ResultBlock, ResultGrant.
- `PolicyDecision` struct: PolicySlug string, Result EvalResult, GrantControls []string, Operator string, SessionControls map[string]interface{}.
- `CombinedDecision` struct: Result EvalResult, GrantControls []string, SessionControls map[string]interface{}, MatchingPolicies []string.
- `ScenarioResult` struct: ScenarioName string, Passed bool, Expected ExpectedOutcome, Got CombinedDecision, MatchingPolicies []string, Error string (optional).
- `TestReport` struct: Files []FileResult. `FileResult` struct: File string, Scenarios []ScenarioResult, Passed/Failed/Errors int.

- Validation: expect.result must be one of "block", "grant", "notApplicable". Return descriptive errors.

Create `internal/testengine/parse_test.go`:

<task type="auto">
  <name>Task 2: Condition matchers with include/exclude semantics</name>
  <files>internal/testengine/match.go, internal/testengine/match_test.go</files>
  <action>
Create `internal/testengine/match.go` with condition matchers. Reuse the pattern from `internal/validate` (getNestedValue, getStringSlice, splitPath) -- copy these as unexported helpers into the testengine package to avoid import coupling.

Each matcher takes the policy's conditions map (map[string]interface{}) and a *SignInContext, returns bool.

Implement these matchers:

1. `matchUsers(conditions, ctx)` -- Check includeUsers/excludeUsers, includeGroups/excludeGroups, includeRoles/excludeRoles. "All" in includeUsers matches all users. "GuestsOrExternalUsers" in includeUsers matches if ctx.User == "guest". Include logic: user is included if they match ANY include list (users OR groups OR roles). Exclude logic: user is excluded if they match ANY exclude list. Pattern: include first, then exclude overrides.

2. `matchApplications(conditions, ctx)` -- Check includeApplications/excludeApplications. "All" matches all apps. Literal GUID comparison otherwise.

3. `matchClientAppTypes(conditions, ctx)` -- Check clientAppTypes array. If absent or empty, match all. If present, ctx.ClientAppType must be in the list. Also match if list contains "all".

4. `matchPlatforms(conditions, ctx)` -- Check platforms.includePlatforms/excludePlatforms. If no platforms block, match all. "all" in includePlatforms matches everything. If ctx.Platform is empty, match (unspecified = any).

5. `matchLocations(conditions, ctx)` -- Check locations.includeLocations/excludeLocations. "All" matches all. "AllTrusted" matches if ctx.Location == "trusted". If no locations block, match all.

6. `matchSignInRiskLevels(conditions, ctx)` -- Check signInRiskLevels array. If empty/absent, match all (no filter). If present, ctx.SignInRiskLevel must be in list.

7. `matchUserRiskLevels(conditions, ctx)` -- Same pattern as signInRiskLevels but for userRiskLevels.

8. Helper: `matchStringList(includeList, excludeList, values []string) bool` -- shared logic per research code example.

CRITICAL PITFALLS (from research):
- Missing condition = matches all (NOT "no match")
- "All" is a keyword, not a GUID
- Exclude runs AFTER include
- Empty risk level arrays = no filter (match all)

Create `internal/testengine/match_test.go`:
- Table-driven tests for each matcher covering: "All" keyword, specific GUID match, exclude override, missing condition block, empty arrays, GuestsOrExternalUsers.
- At least 3-4 test cases per matcher.
  </action>

  <done>All condition matchers pass tests including edge cases: "All" keyword, exclude overrides, missing conditions default to match, empty risk arrays match all</done>
