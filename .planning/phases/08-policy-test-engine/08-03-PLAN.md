  <action>
Create `internal/testengine/runner.go`:

- `LoadPolicies(policyDir string) ([]PolicyWithSlug, error)` -- Read all .json files from policyDir, unmarshal each into map[string]interface{}, return as []PolicyWithSlug where slug = filename without .json. This is simpler than importing cmd.ReadDesiredPolicies (which returns reconcile.BackendPolicy) and avoids a cmd->internal circular dep.

- `RunTests(testPaths []string, policyDir string) (*TestReport, error)`:
  1. Call LoadPolicies to get all policies
  2. For each test path, call RunTestFile
  3. Aggregate into TestReport
  4. Return report

- `RunTestFile(path string, allPolicies []PolicyWithSlug) (*FileResult, error)`:
  1. ParseTestFile(path)
  2. Filter policies if spec.Policies is non-empty (match by slug prefix or exact match)
  3. For each scenario: build SignInContext, call EvaluateAll, compare CombinedDecision against scenario.Expect
  4. Comparison logic:
     - result must match (block/grant/notApplicable mapped from CombinedDecision.Result)
     - If expect.Controls specified, verify all expected controls are present in CombinedDecision.GrantControls
     - SessionControls comparison deferred (omit for now, mark as TODO)
  5. Build ScenarioResult with pass/fail and details

- `resultString(r EvalResult) string` -- convert enum to "block"/"grant"/"notApplicable" for comparison.

Create `internal/testengine/runner_test.go`:
- Test with in-memory policies and test specs (not file I/O)
- Test cases: all pass, one fail, policy filter works, empty policy dir error
- Use testify/assert

Create `internal/testengine/report.go`:

- `RenderHuman(w io.Writer, report *TestReport, useColor bool)`:
  Format per research example:
  ```
  === cactl test ===

  tests/block-legacy-auth.yaml
    PASS  Legacy client should be blocked (N policies matched, result: block)
    FAIL  Modern browser should not be blocked
          expected: notApplicable
          got:      grant [mfa]
          matching policies: cap100-admin-...

  Results: N passed, N failed, N errors
  ```
  Use green for PASS, red for FAIL. Text prefixes when color disabled (consistent with HumanRenderer pattern).

- `RenderJSON(w io.Writer, report *TestReport) error`:
  JSON output with summary (total, passed, failed, errors) and files array with scenario details.

- `Summary(report *TestReport) (total, passed, failed, errors int)` -- compute totals across all files.

Create `cmd/test.go`:

- Register `testCmd` as a subcommand of rootCmd (add `rootCmd.AddCommand(testCmd)` in init()).
- Command: `cactl test [test-files...] --tenant <id> [--output json]`
- Use: "test", Short: "Run policy test scenarios", Long description explaining it evaluates YAML test files against local policy files.

- Flags:
  - `--tenant` (required, from global config/flag -- use viper)
  - `--output` (from global config -- "human" or "json")
  - No auth needed (local-only evaluation, no Graph API calls)

- RunE logic:
  1. Get tenant from viper (same as other commands: `viper.GetString("tenant")`)
  2. If tenant empty, return ExitError code 2 with message
  3. Determine test file paths:
     - If args provided, use those paths directly
     - If no args, discover from `tests/<tenantID>/*.yaml` using filepath.Glob
     - If no test files found, return ExitError code 2 "no test files found"
  4. Policy dir: `policies/<tenantID>/`
  5. Call `testengine.RunTests(testPaths, policyDir)`
  6. Render output: check viper "output" for "json" vs human. Call report.RenderHuman or report.RenderJSON to os.Stdout.
  7. Exit codes: ExitSuccess (0) if all pass, ExitChanges (1) if any fail, ExitFatalError (2) on errors.

IMPORTANT: Do NOT use NewPipeline -- test command does NOT need auth, graph client, or git backend. It only reads files from disk. This keeps it fast and offline.
