<feature>
  <name>CA Policy Evaluation Engine</name>
  <files>internal/testengine/evaluate.go, internal/testengine/evaluate_test.go</files>
  <behavior>
  EvaluatePolicy(policy map[string]interface{}, ctx *SignInContext) -> PolicyDecision:
    - state == "disabled" -> ResultNotApplicable
    - state == "enabledForReportingButNotEnforced" -> evaluate normally (treat as enabled)
    - state == "enabled" -> evaluate conditions
    - All conditions must match (AND): users AND apps AND clientAppTypes AND platforms AND locations AND signInRisk AND userRisk
    - If any condition does not match -> ResultNotApplicable
    - If all conditions match, check grantControls.builtInControls:
      - Contains "block" -> ResultBlock
      - Otherwise -> ResultGrant with controls list and operator from grantControls.operator
    - Session controls extracted from sessionControls block if present

  EvaluateAll(policies []PolicyWithSlug, ctx *SignInContext) -> CombinedDecision:
    - Evaluate each policy independently
    - Skip ResultNotApplicable policies
    - If ANY policy returns ResultBlock -> CombinedDecision.Result = ResultBlock
    - If no block, collect all grant controls from all matching grant policies
    - MatchingPolicies lists slugs of all policies that matched (not NotApplicable)
    - If no policies match -> ResultNotApplicable

  Test cases (input -> expected):
    - Disabled policy, any context -> NotApplicable
    - Block policy matching context -> Block with "block" in controls
    - Grant policy matching context -> Grant with MFA control
    - Policy with non-matching user -> NotApplicable
    - Two policies: one grants MFA, one blocks -> Block wins
    - Two grant policies -> combined controls from both
    - Report-only policy -> evaluated as if enabled
    - Policy with no platform condition, context has platform -> matches (missing = all)
  </behavior>
  <implementation>
Create evaluate_test.go FIRST with table-driven tests covering all cases above. Use inline map[string]interface{} for policy JSON (mimicking real policy structure from the sample files). Run tests -- they must FAIL (RED).

- `EvaluateAll`: iterate policies, collect decisions, apply "block wins" rule, merge grant controls.

Use the JSON traversal helpers (getNestedValue, getStringSlice) already in match.go.

Run tests -- they must PASS (GREEN).

Refactor if needed, ensuring tests still pass.
  </implementation>
</feature>
