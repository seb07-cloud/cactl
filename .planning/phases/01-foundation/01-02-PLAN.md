<task type="auto">
  <name>Task 2: Create ClientFactory with per-tenant credential isolation and unit tests</name>
  <files>
    internal/auth/factory.go

    internal/auth/factory_test.go

  <action>
    Create `internal/auth/factory.go`:
    - Define ClientFactory struct with:
      - provider AuthProvider (the resolved credential provider)
      - credentials map[string]azcore.TokenCredential (keyed by tenantID, for caching)
      - mu sync.RWMutex (protect concurrent access to credentials map)
    - Constructor: NewClientFactory(cfg types.AuthConfig) (*ClientFactory, error):
      - Call ResolveAuthMode(cfg) to determine mode
      - Based on mode, create the appropriate provider:
        - "az-cli" -> &AzureCLIProvider{}
        - "client-secret" -> NewClientSecretProvider(cfg.ClientID, cfg.ClientSecret)
        - "client-certificate" -> NewClientCertificateProvider(cfg.ClientID, cfg.CertPath)
        - unknown -> return error
      - Validate required fields: client-secret needs ClientID+ClientSecret, client-certificate needs ClientID+CertPath
      - Return &ClientFactory{provider: provider, credentials: map}
    - Implement Credential(ctx, tenantID):
      - RLock, check if credential exists in cache for tenantID, RUnlock
      - If cached, return it
      - Lock, double-check (race), create new credential via provider.Credential(ctx, tenantID)
      - Store in cache, Unlock, return
    - Implement Mode() string: delegate to provider.Mode()
    - CRITICAL (AUTH-05): Each tenant gets its OWN credential instance. The cache key is the tenantID. Never share credentials across tenants. This prevents the azidentity token cache bug (#19726).

    Create `internal/auth/provider_test.go`:
    - Test ResolveAuthMode with table-driven tests:
      - Empty config -> "az-cli" (fallback)
      - Mode explicitly set to "client-secret" -> "client-secret"
      - ClientID + ClientSecret set, no explicit mode -> "client-secret" (auto-detect)
      - ClientID + CertPath set, no explicit mode -> "client-certificate" (auto-detect)
      - ClientID + ClientSecret + CertPath set, no explicit mode -> "client-secret" (secret wins)
      - Explicit mode overrides auto-detect (mode="az-cli" with ClientID+ClientSecret set -> "az-cli")

    Create `internal/auth/factory_test.go`:
    - Test NewClientFactory:
      - With valid az-cli config -> factory created, Mode() returns "az-cli"
      - With client-secret config missing ClientID -> error
      - With client-secret config missing ClientSecret -> error
      - With client-certificate config missing CertPath -> error
      - With unknown mode -> error
    - Test per-tenant isolation:
      - Create factory with az-cli mode
      - Call Credential with tenantID "tenant-a" and "tenant-b"
      - Verify both return without error (actual credential creation requires Azure; test the factory logic, not azidentity)
      - NOTE: Cannot fully test credential creation without Azure credentials. Test the factory structure, validation, and mode resolution. Use interface mocking if helpful.

    Install testify: `go get github.com/stretchr/testify@latest`
