**Fix 1 — main.go: Print ExitError.Message to stderr before os.Exit**

In main.go, after `errors.As(err, &exitErr)` succeeds, add:
```go
fmt.Fprintln(os.Stderr, "Error: "+exitErr.Message)
```
before calling `os.Exit(exitErr.Code)`. Also add the same stderr print for the generic error fallback path (non-ExitError), printing `err.Error()` before `os.Exit(types.ExitFatalError)`.

Import `"fmt"` (already imported by some paths, but verify).

**Fix 2 — cmd/root.go: Call config.Load + config.Validate in initConfig**

At the end of the `initConfig` function (after BindPFlags), add:
```go
cfg, err := config.Load(v)
if err != nil {
    return fmt.Errorf("loading config: %w", err)
}
if err := config.Validate(cfg); err != nil {
    return err
}
```

Add import for `"github.com/seb07-cloud/cactl/internal/config"`.

IMPORTANT: The `init` subcommand must still work when no config file exists (pre-init). The current code already handles ConfigFileNotFoundError gracefully -- config.Load will just return defaults from viper when no file is found. Validate will pass on defaults ("human" output, "info" log level, empty auth mode). This is safe.

Do NOT change SilenceErrors -- it must remain true to prevent cobra from double-printing errors. The stderr output in main.go is the correct single place for error display.
