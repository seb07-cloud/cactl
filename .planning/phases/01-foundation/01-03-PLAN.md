    Create `internal/schema/schema.json`:
    - Create a minimal but correct JSON Schema (draft-07) for the conditionalAccessPolicy resource based on the documented properties in the research file.
    - Include the mutable properties: displayName (string, required), state (enum: enabled|disabled|enabledForReportingButNotEnforced, required), conditions (object, required), grantControls (object), sessionControls (object).
    - Do NOT include read-only properties (id, createdDateTime, modifiedDateTime, templateId) in `required` -- they are server-managed.
    - Include nested type stubs for conditionalAccessConditionSet, conditionalAccessGrantControls, conditionalAccessSessionControls with `type: object` and `additionalProperties: true` (these will be fleshed out when the full schema is fetched).
    - This is the embedded fallback used when network fetch fails.

    Create `internal/schema/embedded.go`:
    - Use Go's `embed` package to embed schema.json into the binary
    - Export var EmbeddedSchema []byte via `//go:embed schema.json`
    - Export func WriteEmbedded(path string) error that writes EmbeddedSchema to the given path

    Create `internal/schema/fetch.go`:
    - Export func Fetch(destPath string) error:
      - Attempt to download the CA policy schema from the pre-sliced OpenAPI spec:
        `https://raw.githubusercontent.com/microsoftgraph/msgraph-metadata/master/openapi/v1.0/openapi.yaml`
      - IMPORTANT: The full file is ~30MB. Use an HTTP GET with a timeout of 30 seconds.
      - If download succeeds, extract the `components/schemas/microsoft.graph.conditionalAccessPolicy` definition and related nested types.
      - Convert the OpenAPI schema fragment to a standalone JSON Schema (draft-07).
      - Write to destPath.
      - If download or parsing fails, return the error (caller handles fallback).
    - NOTE: The OpenAPI extraction is complex. For Phase 1, implement a simpler approach:
      - Try to fetch a smaller file first. If the full YAML is too large to parse in init, use the CSDL endpoint or simply fall back to embedded.
      - The PRAGMATIC approach for Phase 1: attempt fetch, if it fails for any reason, return error so caller uses embedded fallback. The fetch logic can be enhanced in later phases.
      - At minimum, implement the HTTP fetch attempt and graceful error return. The embedded fallback ensures init always succeeds.

  <name>Task 2: Implement cactl init command with workspace scaffolding and tests</name>
  <files>

  <action>
    Create `cmd/init.go`:
    - Register initCmd as a subcommand of rootCmd in init()
    - Define initCmd: Use "init", Short "Initialize a cactl workspace", RunE: runInit
    - Implement runInit(cmd, args) following this EXACT order (CONF-03 safety):

      Step 1: Check if .cactl directory already exists.
      - If exists, return ExitError{Code: ExitValidationError, Message: "workspace already initialized (.cactl directory exists)"}

      Step 2: Check if .cactl/config.yaml is tracked by Git (CONF-03).
      - Run `git ls-files --error-unmatch .cactl/config.yaml` via os/exec
      - If exit code 0 (file IS tracked), return ExitError{Code: ExitValidationError, Message: ".cactl/config.yaml is tracked by Git -- remove it first: git rm --cached .cactl/config.yaml"}
      - If exit code non-zero (not tracked), continue (this is the expected path)
      - If git is not available, skip this check silently (workspace may not be a git repo)

      Step 3: Create .cactl directory with 0755 permissions.

      Step 4: Update .gitignore BEFORE creating config.yaml (CONF-03 order).
      - If .gitignore exists, read it. Check if ".cactl/config.yaml" line already present. If not, append "\n# cactl workspace\n.cactl/config.yaml\n".
      - If .gitignore does not exist, create it with "# cactl workspace\n.cactl/config.yaml\n".

      Step 5: Write .cactl/config.yaml with default content (CONF-01).
      - Template:
        ```yaml
        # cactl configuration
        # All values can be overridden by CACTL_* environment variables or CLI flags
        # See: https://github.com/seb07-cloud/cactl
        tenant: ""
        auth:
          mode: ""  # az-cli | client-secret | client-certificate (auto-detected if empty)
        output: human  # human | json
        log_level: info
        ```
      - Note: NO secrets in this file. ClientID, ClientSecret, CertPath come from env vars only.

      Step 6: Fetch CA policy JSON Schema (CONF-04).
      - Call schema.Fetch(".cactl/schema.json")
      - If error, log warning (not fatal): "Warning: could not fetch CA policy schema: %v"
      - On failure, call schema.WriteEmbedded(".cactl/schema.json") for fallback
      - If even embedded write fails, return error (this is fatal)

      Step 7: Output success using the renderer from the output package.
      - Use human or json output based on --output flag
      - Show: "Workspace initialized in .cactl/"
      - Show: files created (config.yaml, schema.json, .gitignore updated)

    Create `cmd/init_test.go`:
    - Test in a temporary directory (t.TempDir())
    - Test cases:
      1. Happy path: init in empty dir -> .cactl/config.yaml exists, .cactl/schema.json exists, .gitignore contains ".cactl/config.yaml"
      2. Already initialized: create .cactl dir first -> init returns ExitValidationError
      3. .gitignore append: create existing .gitignore with other content -> init appends cactl entry without overwriting
      4. .gitignore idempotent: .gitignore already has ".cactl/config.yaml" -> init does not duplicate the entry
      5. Config content: verify .cactl/config.yaml is valid YAML with expected keys (tenant, auth.mode, output, log_level)
    - Use os.Chdir to the temp dir for each test (restore after via t.Cleanup)
    - Do NOT test Git tracking check in unit tests (requires Git repo setup) -- that's an integration test concern
