<task type="auto">
  <name>Task 2: Create root command with global flags, Viper config loading, and output renderer</name>
  <files>
    cmd/root.go
    internal/config/config.go
    internal/config/validate.go
    internal/output/renderer.go

  <action>
    Create `cmd/root.go`:
    - Define rootCmd as &cobra.Command with Use: "cactl", Short description, SilenceUsage: true, SilenceErrors: true
    - Set PersistentPreRunE to call initConfig(cmd)
    - In init(), register PersistentFlags on rootCmd:
      - --tenant (string, "Entra tenant ID or primary domain")
      - --output (string, default "human", "Output format: human|json")
      - --no-color (bool, "Disable ANSI color output")
      - --ci (bool, "Non-interactive CI mode")
      - --config (string, "Config file path, default .cactl/config.yaml")
      - --log-level (string, default "info", "Log level: debug|info|warn|error")
      - --auth-mode (string, "Auth mode: az-cli|client-secret|client-certificate")
    - Implement initConfig(cmd) following research Pattern 1 exactly:
      - Read --config flag for custom config path; default to .cactl/config.yaml
      - Set viper env prefix to "CACTL", call AutomaticEnv()
      - Set env key replacer: strings.NewReplacer("-", "_", ".", "_")
      - ReadInConfig (ignore ConfigFileNotFoundError, return other errors)
      - BindPFlags(cmd.Flags()) AFTER config is read (critical ordering per research Pitfall 3)
    - Export Execute() function that returns rootCmd.Execute()

    Create `internal/config/config.go`:
    - func Load(v *viper.Viper) (*types.Config, error):
      - Unmarshal viper into types.Config using mapstructure
      - Override auth secrets from viper (not config file): ClientID from "client_id", ClientSecret from "client_secret", CertPath from "cert_path"
      - Return loaded config
    - func LoadFromGlobal() (*types.Config, error) convenience wrapper using viper.GetViper()

    Create `internal/config/validate.go`:
    - func Validate(cfg *types.Config) error:
      - Validate Output is "human" or "json" (return ExitError code 3 if invalid)
      - Validate LogLevel is one of debug|info|warn|error (return ExitError code 3 if invalid)
      - Validate AuthMode if set is one of az-cli|client-secret|client-certificate (return ExitError code 3 if invalid)

    Create `internal/output/color.go`:
    - func ShouldUseColor(v *viper.Viper) bool:
      - Return false if v.GetBool("no-color") is true
      - Return false if os.Getenv("NO_COLOR") != "" (no-color.org convention)
      - Return false if v.GetBool("ci") is true
      - Return false if stdout is not a terminal (check os.Stdout.Stat() for ModeCharDevice)
      - Return true otherwise

    Create `internal/output/renderer.go`:
    - Define Renderer interface with methods:
      - Success(msg string)
      - Error(msg string)
      - Info(msg string)
      - Warn(msg string)
      - Print(msg string)
    - func NewRenderer(format string, useColor bool) Renderer:
      - If format == "json", return &JSONRenderer{}
      - Otherwise return &HumanRenderer{useColor: useColor}

    Create `internal/output/human.go`:
    - Implement HumanRenderer struct with useColor bool field
    - Implement all Renderer methods, prefixing with colored symbols when useColor is true:
      - Success: green checkmark
      - Error: red cross
      - Info: blue info marker
      - Warn: yellow warning marker
      - Print: no prefix

    Create `internal/output/json.go`:
    - Implement JSONRenderer struct
    - Implement all Renderer methods outputting JSON objects to stdout:
      - {"level": "success", "message": "..."}
      - {"level": "error", "message": "..."}
      - etc.

    Install dependencies: `go get github.com/spf13/cobra@latest github.com/spf13/viper@latest`
    Run `go mod tidy`.
  </action>

    `go vet ./...` passes.
  </verify>
  <done>
