**Step A: Consolidate historyEntry (DUP-12)**

Both history.go and status.go define nearly identical `historyEntry` structs for JSON output. history.go's version has a `Changes` field that status.go's doesn't.

1. In cmd/history.go, move the `historyEntry` struct to package-level (not inside a function):
```go
// historyEntry is the JSON structure for version history entries.
// Used by both history and status --history commands.
type historyEntry struct {
    Version   string `json:"version"`
    Timestamp string `json:"timestamp"`
    Message   string `json:"message"`
    Changes   string `json:"changes,omitempty"`
}
```
Note the `omitempty` on Changes -- this way status.go can use the same struct without populating Changes, and the JSON output remains identical (no empty "changes" field).

2. In cmd/status.go's runStatusHistory function, remove the local historyEntry struct definition and use the shared one from history.go. Since both files are in package cmd, this works directly.
